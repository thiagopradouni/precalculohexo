Comparação de Eficiência dos Algoritmos de Ordenação

Nesta seção, comparo a eficiência dos algoritmos Bubble Sort, Selection Sort, Insertion Sort e Merge Sort com base em sua complexidade temporal e desempenho.

Desempenho Prático

Os algoritmos foram testados com arrays de diferentes tamanhos (100, 2.500, 5.000, 7.500, 10.000, 12.500, 15.000) e os tempos de execução foram registrados. Abaixo, um resumo dos resultados:

Tempo de Execução (em segundos)

Tamanho do Array Bubble Sort Selection Sort Insertion Sort Merge Sort
100 0,003 0,003 0,001 0,001
2.500 1,232 0,687 0,700 0,026
5.000 5,027 2,650 2,864 0,064
7.500 11,258 6,229 6,011 0,088
10.000 20,281 10,902 10,600 0,121
12.500 31,986 16,424 16,756 0,156
15.000 45,285 23,88 24,420 0,191

Análise de Eficiência

  • Bubble Sort: Seu tempo de execução cresce rapidamente com o aumento do tamanho do array, tornando-o impraticável para grandes listas.
  • Selection Sort: Embora melhor que Bubble Sort, ainda é ineficiente para listas grandes devido à sua complexidade O(n²).
  • Insertion Sort: Melhor desempenho em listas parcialmente ordenadas, mas ainda ineficiente para listas grandes desordenadas.
  • Merge Sort: Consistentemente eficiente com complexidade O(n log n), ideal para grandes conjuntos de dados, mas consome mais memória.

Conclusão

Para pequenas listas ou listas quase ordenadas, Insertion Sort pode ser eficiente. Para listas desordenadas ou grandes, Merge Sort é a escolha ideal devido à sua eficiência consistente. Bubble Sort e Selection Sort são mais adequados para fins educacionais ou listas muito pequenas, devido à sua ineficiência em listas maiores.
